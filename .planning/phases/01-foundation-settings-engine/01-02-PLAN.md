2. **src/config/merger.ts** -- Pure merge function (from RESEARCH.md Pattern 2):
   - `MergeInput`: { existing: ClaudeSettings; newHooks: { event: string; matcher?: string; hook: HookEntry }[] }
   - `mergeHooks(input: MergeInput): MergeResult`
   - Algorithm:
     a. structuredClone(input.existing) -- never mutate input
     b. Ensure settings.hooks exists (default to {})
     c. For each newHook: find matching event array, check for exact duplicate (same event + matcher + command string) -> skip if found
     d. If no duplicate: append a new HookGroup { matcher?, hooks: [{ type, command }] } to the event array
     e. Return { settings, added, skipped }
   - Conflict detection: same event+matcher is a WARNING (logged by caller), not a block. Only exact command duplicates are skipped.
   - CRITICAL: Only touch settings.hooks. All other keys pass through via structuredClone.
