**src/commands/init.ts** -- Replace the stub from Plan 01:

```typescript
export async function initCommand(opts: { scope: string; dryRun?: boolean }): Promise<void>
```

Implementation flow:
1. Resolve scope from opts.scope (validate it's 'user'|'project'|'local', default 'project')
2. Get settingsPath via getSettingsPath(scope)
3. Get hooksDir via getHooksDir(scope)
4. Read existing settings via readSettings(settingsPath)
5. **Already initialized check:** If settings.hooks exists and is non-empty, print "Already initialized at {settingsPath}" (using log.info) and return. Per locked decision: no-op on re-init.
6. If --dry-run:
   - Print "[DRY RUN] Would create directory: {hooksDir}" (if not exists)
   - Print "[DRY RUN] Would seed settings.json at: {settingsPath}"
   - Print "[DRY RUN] Would add empty hooks structure"
   - Return without writing
7. Create hooksDir: `mkdir -p` equivalent (fs.mkdir with recursive: true)
8. Create parent directory for settings if needed: `mkdir -p` on dirname of settingsPath
9. Use applyMerge with empty newHooks array -- this seeds the hooks key as {} while preserving everything else. Actually, applyMerge with empty hooks won't add the hooks key. Instead:
   - Read existing settings
   - If no hooks key: create backup, write settings with `hooks: {}` added (use writeSettings directly)
   - If hooks key exists but empty: same as above (no-op, already seeded)
10. Print summary:
    - "Created {hooksDir}" (if directory was created)
    - "Seeded {settingsPath} with hooks configuration" (if settings were modified)
    - "Backed up existing settings to {backupPath}" (if backup was created)

Note: init does NOT install any hooks -- it just creates the directory and empty hooks structure. Hook installation is Phase 2's `add` command.

For tests: mock the filesystem using tmp directories. Override the locator paths by either:
- Passing settingsPath/hooksDir explicitly to an internal function
- Or using a test helper that sets up the correct directory structure in tmpdir and runs init against it
