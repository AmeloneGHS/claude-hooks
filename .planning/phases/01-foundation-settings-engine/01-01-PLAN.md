<task type="auto">
  <name>Task 2: TypeScript types, Zod schemas, Commander.js CLI skeleton, logger utility</name>
  <files>src/cli.ts, src/types/settings.ts, src/types/hooks.ts, src/registry/types.ts, src/utils/logger.ts, src/commands/init.ts, src/commands/restore.ts</files>
  <action>
1. **src/types/settings.ts** -- Claude Code settings.json TypeScript types:
   - `HookEntry`: { type: 'command'; command: string }
   - `HookGroup`: { matcher?: string; hooks: HookEntry[] }
   - `ClaudeSettings`: { env?: Record<string, string>; hooks?: Record<string, HookGroup[]>; mcpServers?: Record<string, unknown>; enabledPlugins?: Record<string, boolean>; statusLine?: unknown; [key: string]: unknown } -- use index signature so unknown future keys pass through
   - `MergeResult`: { settings: ClaudeSettings; added: { event: string; matcher?: string; command: string }[]; skipped: { event: string; matcher?: string; command: string; reason: string }[] }

2. **src/types/hooks.ts** -- Hook event types and Zod schemas:
   - `HookEvent` union type with all 25 events from RESEARCH.md (PreToolUse, PostToolUse, PostToolUseFailure, PermissionRequest, SessionStart, Stop, StopFailure, SessionEnd, Notification, SubagentStart, SubagentStop, UserPromptSubmit, ConfigChange, PreCompact, PostCompact, WorktreeCreate, WorktreeRemove, TeammateIdle, TaskCompleted, InstructionsLoaded, Elicitation, ElicitationResult)
   - `hookEventSchema` Zod enum covering all events
   - `hookEntrySchema` Zod object: { type: z.literal('command'), command: z.string() }
   - `hookGroupSchema` Zod object: { matcher: z.string().optional(), hooks: z.array(hookEntrySchema) }
   - `claudeSettingsSchema` Zod object with hooks as optional record of string -> array of hookGroupSchema, plus passthrough() for unknown keys

3. **src/registry/types.ts** -- Registry data model (from RESEARCH.md):
   - `HookDefinition`: { name, description, event: HookEvent, matcher?: string, pack?: string, scriptFile: string }
   - `HookPack`: { name, description, hooks: string[] }
   - `Registry`: { hooks: Record<string, HookDefinition>, packs: Record<string, HookPack> }
   - Zod schemas for each: `hookDefinitionSchema`, `hookPackSchema`, `registrySchema`

4. **src/utils/logger.ts** -- Colored output helpers using picocolors:
   - `log.info(msg)` -- default color
   - `log.success(msg)` -- green
   - `log.warn(msg)` -- yellow
   - `log.error(msg)` -- red
   - `log.dim(msg)` -- gray/dim
   - `log.dryRun(msg)` -- yellow "[DRY RUN]" prefix

5. **src/cli.ts** -- Commander.js entry point:
   - Import Commander, set name('claude-hooks'), description, version (read from package.json using createRequire or hardcode '0.1.0' for now)
   - Register 'init' command with --scope (default 'project') and --dry-run options, lazy import from './commands/init.js'
   - Register 'restore' command with --scope option, lazy import from './commands/restore.js'
   - Call program.parse()

6. **src/commands/init.ts** -- Stub:
   - Export `initCommand(opts: { scope: string; dryRun?: boolean })` that logs "Init not yet implemented" and exits

7. **src/commands/restore.ts** -- Stub:
   - Export `restoreCommand(opts: { scope: string })` that logs "Restore not yet implemented" and exits

After creating all files, run build + typecheck to confirm everything compiles.
